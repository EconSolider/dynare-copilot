# 流程详解（主流程各步的展开）

> **何时读**：执行 SKILL.md §4 主流程时，需要某一步的细节就读对应小节。本文件是
> SKILL.md 的展开，不重复其精简版要点。

## 第1.5步　逐主体确认特征（⏸ 暂停点）

在搭建任何经济主体前，先就模型含哪些主体、各主体什么特征**逐一问用户**，不要默认按某个
"标准版本"写——主体设定写错往往要整段重写。

**⏸ 暂停点**：把问题作为可见文本输出给用户 → 结束本轮回复 → 等用户答。不要在内部替用户
拟答案、不要反复斟酌组合、不要跳过提问继续写。

**何时跳过**（直接进第2步）：用户已点名论文（走第3步检索）、已给完整方程组、或本轮已把
主体特征讲清楚。

**怎么问**：先列计划包含的主体请用户增删，再对每个纳入主体问关键特征，**一条消息集中问完**
（用选项让用户快速选）。各主体常见待定特征：

```
家庭 Households：
  - 效用：可分/不可分劳动？GHH 还是 KPR？是否含消费习惯(habit)？
  - 是否异质（储蓄者 vs 手停口停 / HANK）？是否含 Epstein-Zin 偏好？
  - 劳动供给：内生小时数？是否含工资黏性(工会)？
厂商 Firms：
  - 市场结构：完全竞争还是垄断竞争？是否分最终品/中间品两层？
  - 价格黏性：Calvo 还是 Rotemberg？是否指数化？是否含工资黏性？
  - 生产要素：资本+劳动？是否含可变资本利用率、投资调整成本、固定成本？
中央银行 Central bank：
  - 规则：泰勒规则(对通胀/产出缺口/利率平滑如何反应)？货币增长规则？最优政策(Ramsey/discretion)？
  - 是否含 ZLB/有效下限？盯通胀还是物价水平？
政府 Government / 财政：
  - 政府支出冲击？税收（一次总付 vs 扭曲性：劳动/资本/消费税）？债务与财政规则？预算如何平衡？
银行/金融 Banks / Financial：
  - 金融摩擦（BGG 金融加速器 / Gertler-Karadi 银行净值 / 抵押约束）？信贷利差、银行资本约束、流动性？
其他：开放经济（汇率、UIP、贸易）？外生过程有哪些冲击及其持续性？数据频率（季度/年度）？
```

确认结果写进文件头注释（中文）作为方程依据。信息不全宁可⏸停下提问，不默默假设；用户说
"你看着办"时给清晰默认建议并声明"按此默认进行，如需调整请说"再继续。

## 第3步　知识判定 + 文献检索

**触发条件**（任一成立 → 执行文献检索再动笔）：
- 用户点名某论文/模型（"复制 Bianchi 2011"）而对其方程组、校准或时序没把握；
- 模型含不熟悉的机制（特殊摩擦、约束、偏好、市场结构），凭记忆写不出完整自洽方程组；
- 用户提供的方程不完整，需原文补全；
- "记得大概"但说不出每条 FOC——**"大概记得" = 不会，必须搜**。

不触发（教科书标准件且方程组完全有把握，如基础 RBC、三方程 NK）→ 直接第4步。

**文献检索**——凭模糊记忆写不熟的模型是最危险的失误来源。**触发即实际调用 web 搜索工具
产出查询**，不要"在脑子里回忆论文"当检索。按可信度分级：

```
a. 先搜现成 Dynare 实现（最可信，可直接当模板）：
   - DSGE_mod：  "<作者> <年份> site:github.com DSGE_mod"
   - 其他复制：  "<论文名> dynare mod file replication"（含 Macro Model Data Base）
   - 官方论坛：  "<模型名> site:forum.dynare.org"
b. 没有现成实现 → 搜论文原文与技术附录：
   - "<作者> <年份> <论文名> working paper pdf"（NBER/SSRN/期刊/作者主页）
   - 完整 FOC、稳态推导、校准表常在 Online/Technical Appendix 而非正文。
c. 必须提取四样（缺一不可，缺了继续搜或问用户）：
   ① 完整均衡方程组（逐条 FOC + 约束 + 外生过程）
   ② 校准/估计参数表（含频率：季度/年度）
   ③ 时序设定（资本期末还是期初存量？预期算子信息集？）
   ④ 冲击设定（哪些冲击、标准差、相关性）
d. 可信度纪律：论文原文/附录 > 作者代码 > 知名复制(DSGE_mod) > 论坛 > 博客/课件。
   - .mod 与论文不一致 → 以论文为准，文件头注明差异；
   - 检索后仍缺关键信息 → ⏸ 停下问用户（要 PDF/版本），不编造方程或校准值填空；
   - 文件头注释列出实际依据的来源。
无联网工具时：⏸ 跳过检索，告诉用户缺什么、需提供什么（PDF/方程组/校准表），不硬写。
```

> 检索本身即可见产出。不要把"知识判定"变成长时间无产出的内部独白：要么立刻判为有把握
> 进第4步，要么立刻搜索/提问。

## 第4步　增量构建（先推导 → 列变量 → 写方程 → 稳态 → 实验；每阶段跑通/确认才往下写）

**绝不一次写完整个文件再跑**——后面的错误会和前面的纠缠难定位。分阶段递进，阶段3起每阶段用
「§4.1 运行与纠错闭环」跑一遍，本阶段过关才写下一阶段。每阶段是独立 TodoList 条目。
形式判定(R8)：**非线性优先**——默认原始非线性方程组；仅 `discretionary_policy` 或用户要线性版
/论文只给线性化系统时写 `model(linear);`。见下方「非线性优先（R8）」。

```
─ 阶段 1：先写推导文件（每次建模必做；⏸ 暂停点）────────────────
  目的：先把经济学做对（数学层面），再机械地翻成 Dynare。错在数学层面就该在这里被你我拦住，
        而不是等跑出错误 IRF。
  产出：单独生成 markdown 文件 <模型名>_derivation.md（不是写进 .mod）。
        **严格按 references/derivation-style.md 的结构模板与公式规范写**（八节固定结构、
        LaTeX 公式书写规则、FOC 编号、变量符号表）。八节为：
        1. 模型概述：是什么模型、来源/文献、本文件做什么实验、含哪些主体。
        2. 主体的最优化问题：逐主体写目标函数 + 预算/技术/资源约束（用 LaTeX 数学）。
        3. 一阶条件 FOC：逐主体推导并列出（欧拉/劳动供给/资本/定价/政策规则等），每条编号 (F1)(F2)…。
        4. 市场出清与总量恒等式：资源约束、要素市场出清、总量加总（异质主体按份额加总）。
        5. 外生过程：各 AR/冲击的形式与持续性。
        6. 稳态求解：稳态方程组 + 解析解/反解步骤，按可自上而下求值的次序排列，
           写到能直接照抄进 steady_state_model；线性化模型注明稳态全 0。
        7. 时序约定：期末/期初存量、是否取对数（形式默认非线性，R8；除非命中 R8 两例外）。
        8. 变量与参数对照表（预告阶段2）：FOC 里出现的内生变量、外生、参数清单，并标注
           每个变量对应哪条 FOC，便于核对方程数=变量数。
           **【教训1 硬查】第8节表必须逐行核对"由哪条方程决定"列：若任一 var 变量该列为空，
           说明该方程缺失，必须补进推导后才能进阶段2，不得留空进入编码阶段。**
           常见易漏：CES/Dixit-Stiglitz 类消费束的**聚合定义式**本身（形如
           $CES_t = [...]^{\varepsilon/(\varepsilon-1)}$）与需求方程（F1/F2）是两条不同方程，
           两者都需列出；只有需求方程而无聚合定义式，`ces` 就是无方程可管的悬空变量。

        **【教训2 Walras 检查】推导第4节写完市场出清后，必须显式执行 Walras 定律检查**：
           在 GE 中，商品/劳动/资产/国际收支等均衡条件中必有一条冗余（Walras 定律）。
           步骤：① 数出所有均衡条件；② 找出哪条等于其余条件的线性组合（最常见：家庭
           预算约束 = 商品市场出清 + NFA 定义 + UIP/国际收支恒等式）；③ 在推导里明确
           注明"该方程由 Walras 定律冗余，**不进 model 块**"，并写出推导路径（如
           '家庭预算约束 ⟵ F17+F20+F8'）。未标注而把冗余方程写进 .mod，表面方程数=
           变量数，实际独立方程少一条，Dynare 特征值爆炸、BK 失败。

  ⏸ 暂停：把推导文件交付用户 → 结束本轮 → 等用户确认或修改。用户点头后才进阶段2。
        若推导中遇到建模分叉（劳动供给同异、含资本否、市场结构），按反空转铁律在此一并问清。
  衔接（核心：以推导 md 为蓝本，对照翻译，不凭记忆重推）：
        阶段2/3/4 写 .mod 时，**全程打开阶段1 的推导 md 逐条对照着写**——变量照第8节清单、
        方程照第2–5节的 FOC、稳态照第6节——而不是从头在脑子里重新组织。这样更快、更准，
        且产出与推导一致。每条 model 方程用 [name='...'] 标注它来自推导里哪条 FOC（编号/论文
        方程号），方程数=变量数(R4)自然对得上。若翻译中发现推导有误，回头修 md 再继续，保持一致。
─ 阶段 2：先列变量清单（只声明、不写方程）──────────────────────
  做 .mod 的第一件事就是把变量理清楚，别一上来就写方程。
  **照推导 md 第8节"变量与参数对照表"逐行翻译成声明**（数学符号 → ASCII 名，如 ψ→psi、
  β→betta，R5）：
  写：文件头注释 + 三类声明：
       var      内生变量（含 long_name；状态量、控制量、辅助 log_x 都在此）
       varexo   外生变量——只放创新项 eps_*(R3)，AR 等持续过程是内生
       parameters 参数（被 steady_state_model 反解的参数也先声明）
  逐一核对：第8节表里每个内生量都已 var；只有创新项进 varexo；参数齐全、无禁用命名(R5)。
  产出：把这份变量清单整理出来（可向用户简述"本模型含哪些内生/外生/参数"）作为方程的基准。
  确认无误 → 进阶段 3 写方程。（此步不必跑 Dynare，是动笔的第一步。）
─ 阶段 3：写模型方程，核对方程数 = 变量数 ────────────────────────
  **照推导 md 第2–5节的 FOC/约束/外生过程逐条翻译成 model 方程**，一条 FOC = 一条方程，
  顺序可照推导编号；每条加 [name='Fk: ...'] 回指来源（R1 注释中文写在上方）。
  写：参数校准 + model 块（依阶段2 的变量清单 + 推导 md 的 FOC 逐条写）。
      暂不写稳态/shocks/实验；末尾临时加空 initval; end; 占位以便跑通。
  看：Dynare 报告的内生变量数 vs 方程数；或 equations/endogenous 不匹配报错。
  过关：方程数 = 内生变量数(R4)，无语法/命名错(R1/R5/R7)、无时序笔误。
       （数目不符时，对照推导 md 第8节表逐条勾：哪条 FOC 漏翻、哪个变量多/少声明。）
  不过 → 查 debugging.md「阶段3：如何核对方程数」与「报错→病因→修法」，修后重跑。
─ 阶段 4：加稳态求解，必须先求出稳态 ─────────────────────────────
  写：删占位，加稳态部分 + resid; steady; check;。
      **首选：把阶段1 推导第6节的稳态解照抄成 steady_state_model**（推导阶段已把解析解/反解
      步骤按自上而下次序写好，这里直接落成代码即可，校准目标用反解）；
      推导未能给出解析解 → initval 给每个内生变量经济合理初猜；仍不收敛 → homotopy_setup
      （steady-state.md）。
  看：resid 残差全近 0；steady 值无 NaN、比率合理(C/Y、I/Y、K/Y)；check 的 BK 结论。
  过关：steady 成功、resid 近 0。（BK 不满足可留到阶段 5。）
  不过 → 查 steady-state.md 与 debugging.md：残差大的方程常暴露稳态代数错或模型笔误
        （若推导的稳态解有误，回头修推导 md，保持两者一致）。
─ 阶段 5：加冲击与实验命令，跑出结果 ─────────────────────────────
  写：shocks（随机用 stderr/var=/corr；确定性用 periods/values）+ 实验命令
      （stoch_simul / perfect_foresight_* / estimation / ramsey_* / osr）。
  看：BK 满足、命令正常产出（政策函数+矩+IRF / 过渡路径 / 后验）。
  过关：实验命令干净跑完、结果合理。
  不过 → BK 不满足优先查 R2/符号/R4；其余按 §4.1 闭环处理。
```

### §4.1 运行与纠错闭环（阶段3起每次跑都走这个）

环境配置（本机路径；用户告知新路径则更新）：

| 环境项 | 路径 / 值 | 说明 |
|--------|-----------|------|
| Dynare 安装目录 | `C:\dynare\7.1` | Dynare 7.1 |
| Dynare matlab 路径 | `C:\dynare\7.1\matlab` | `addpath` 用这个目录 |
| MATLAB 安装目录 | `D:\Program Files\Matlab` | 供 MCP 服务器/启动用 |
| 工作目录 | `.mod` 文件所在文件夹 | 运行前 `cd` 到这里 |

```
0. 前置（一次）：工具列表找 MATLAB MCP（执行 MATLAB 命令、读工作区、捕获输出）。
   找不到 → 告知用户在 VSCode 启用；改为只交付文件+运行说明（仍按分阶段写）。
1. 初始化（每个新 MATLAB 会话一次）：
      addpath('C:\dynare\7.1\matlab');
      cd('<.mod所在目录>');
2. 循环（n=1，上限 5 轮）：
   a. 运行：dynare <文件名不带扩展名> noclearall   （调试期可加 nograph）
   b. 按输出分支：
      - ERROR/syntax error/报错堆栈 → 查 debugging.md「报错→病因→修法」
      - "Blanchard-Kahn conditions are not satisfied" → 优先查 R2/符号/R4；跑 model_info;
      - 稳态含 NaN 或 resid 残差大 → 查 steady-state.md；跑 resid; 定位残差大的方程
      - 干净跑完 → 本阶段过关（到阶段5 干净跑完后，做下面第 3 条"验证"）
   c. 修复纪律：每轮只改一处，说明"改了什么、为什么"；TodoList 追加并勾选"第n轮：修复xx"；
      n=n+1；n>5 或同一错误重复 2 轮 → ⏸ 停止，向用户报告卡点、已试方案、所需信息
      （更好初猜/正确校准目标/数据文件），不无限重试。
3. 验证（阶段5 干净跑完后，读工作区并向用户摘要）：
      oo_.dr.eigval      % BK 特征值
      oo_.steady_state   % 稳态向量：无 NaN、比率合理
      oo_.mean, oo_.var  % 理论矩（随机模拟）
      oo_.irfs           % IRF（如有）
```
调试期可随时通过 MCP 追加：`resid;`、`model_diagnostics;`、`model_info;`、`check;`。

## 第6步　收尾清理（交付前复查工作目录）

迭代过程会留下大量中间产物，交付前复查文件夹并清理。原则：**白名单内的明确中间产物直接删；
来源不确定的先列出来问用户；黑名单内的绝不删**。

**白名单（可直接删）**：
- Dynare 自动生成（以 .mod 文件名 `<m>` 为根）：
  - `+<m>/`（预处理器生成的 MATLAB 包文件夹）；
  - `<m>/`（图形输出夹，含 `graphs/`）；`Output/`（Dynare 5+ 放 _results、log 处）；
  - `<m>_results.mat`、`<m>.log`、`<m>.jnl`；
  - 自动生成的 `.m`：`<m>_dynamic.m`、`<m>_static.m`、`<m>_set_auxiliary_variables.m`、
    `driver.m` 等；`steady_state_model` 块的自动稳态产物在 `+<m>/steadystate.m`，随 `+<m>/`
    一并删（与根目录手写的 `<m>_steadystate.m` 是两回事，详见 steady-state.md
    「手写稳态文件的两种形态」）；
  - IRF/图：`<m>_IRF_*.eps/.pdf/.fig`、`simulation_IRFs/` 等（除非用户要留图）。
- agent 自己留下的临时物：调试脚本、跑崩或被替代的旧版 `.mod`（如 `test.mod`、
  `<m>_v1_broken.mod`）、临时占位文件。

**黑名单（绝不删，即使看着像垃圾也要先问）**：
- 用户上传的任何文件、数据文件（`.mat` 数据、`.xlsx`、`.csv`、观测数据等）；
- 推导 md（`<m>_derivation.md`）、最终 `.mod`、本 skill 的参考文件；
- **用户手写的稳态文件 `<m>_steadystate.m`**（无 `2` 后缀）——用户提供的外部稳态函数，
  **不是** 自动产物，删了模型跑不出稳态。它与自动产物 `+<m>/steadystate.m` 不同名、不同位置
  （区别详见 steady-state.md「手写稳态文件的两种形态」），注意别误删手写的。

**流程**：列出工作目录 → 按白名单识别并直接删 → 无法确定归属的列给用户确认 → 交付时报告
"删了 X、保留了 Y"。清理不是必须跑 MATLAB，用文件操作即可；但删前确认这些不是用户要的产物。

## 最终文件骨架（各阶段都写完后的结构；Dynare 对顺序敏感）

```dynare
/*
 * 文件头（中文）：模型是什么、来源、本文件做什么实验、关键约定（时序、是否取对数）。
 * 各主体特征（第1.5步确认结果）也记在这里。
 */

//==================== 变量声明 ====================
var y ${y}$ (long_name='output') ... ;          // 内生；行尾中文注释
varexo eps_z ${\varepsilon_z}$ (long_name='TFP shock') ... ;  // 仅创新项 (R3)
parameters alppha ${\alpha}$ (long_name='capital share') ... ;

//==================== 参数校准 ====================
alppha = 0.33;
// 被 steady_state_model 反解的参数不在此赋值
```

## DSGE_mod 最佳实践（产出质量标准）

1. 中文文件头块：模型、来源、实验、约定、各主体特征。
2. 变量/参数带 LaTeX 名 + 英文 `long_name`：`var y ${y}$ (long_name='output');`。
3. 每条方程 `[name='...']`（英文，最好引论文方程号）+ 上方中文注释。
4. 校准目标在 `steady_state_model` 内反解（如 `l=0.33` 反解 `psi`）。
5. 加 `log_x = log(x);` 辅助变量用于报告与观测方程。
6. 估计的观测方程优先用增长率（稳态为 0，对趋势稳健）。
7. 变体/重复结构用宏处理器（见 `references/macro-processor.md`）。
8. 流程固定：`resid; steady; check;` 再进实验命令。

## 细则（写文件时按需查阅）

### 时序约定（R2 展开）
- 变量时间下标 = **被决定的期**；"期末存量"约定。
- 控制/跳跃变量当期决定 → 自身定义无超前（可在预期中以 `(+1)` 出现，如欧拉方程 `c(+1)`）。
- 预定/状态变量上期决定 → 当期带滞后：生产 `k(-1)`、运动律 `k = invest + (1-delta)*k(-1)`。
- 超前/滞后写 `x(+1)`、`x(-2)`；参数无时间下标。
- 偏好"期初存量"则 `predetermined_variables k;`，生产写 `k`、运动律 `k(+1)=invest+(1-delta)*k;`。
  二选一并全文一致。

### 非线性优先（R8 展开）
**默认写原始非线性方程组**（FOC/约束/外生过程），交给 Dynare 做泰勒展开，不手推线性化。
理由：手推线性化是隐蔽错误重灾区且 Dynare 检不出；非线性才能做高阶近似（order=2/3：福利、
风险溢价、不确定性冲击）；稳态有经济含义可核对大比率；同一文件可在一阶/二阶/完全预见间切换。

**仅两种情况写线性化**（`model(linear);`，变量是百分比/对数偏离，稳态通常全 0）：
1. `discretionary_policy` 任务——Dynare 技术上要求线性模型；
2. 用户明确要线性版，或复制的论文只给出线性化系统。

除此之外一律非线性，不需要在脑内判断"该不该线性化"。

配套：报告对数偏离 IRF 加 `log_y = log(y);` 辅助变量；**非线性模型冲击标准差按小数写**
（1% = `stderr 0.01`，不乘 100）；AR 过程直接写 `log(A) = rho_a*log(A(-1)) + eps_a;`。

### 语言规则（R1 展开）
- 注释全中文且充分：`//` 行内、`/* */` 块、文件头，解释模型/方程含义/时序选择/参数来源。
- 方程的中文注释放 `[name=...]` **上方单独一行**，不与标注同行混排。
- 声明行可在行尾加简短中文注释（`// 消费`）。
- `long_name`/`[name=]`/标识符/其余字符串全英文 ASCII。

### 模型块要点
- 方程 `左 = 右;`；齐次式可只写一边（隐含 `=0`）。
- `>=`/`<=`/`==` 不是不等式约束（`a>=b` 被读作 `(a>=b)=0`）。
- 模型局部变量 `#z = ...;` 跨方程共享子表达式，作用域仅模型块。
- `STEADY_STATE(x)` 取稳态值（泰勒规则/产出缺口常用）。
- `EXPECTATION(-1)(x(+1))` 取滞后信息集下的预期。
