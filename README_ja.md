> これは Claude Code の skill です。あなたのマクロ経済の直感——「金融アクセラレータ付きのニューケインジアンモデルが欲しい」「Smets-Wouters 2007 を再現して」——を、定常状態が解かれ IRF まで手に入る、動作する検証済みの Dynare `.mod` ファイルへと変換します。
>
> その核心は、経験豊富なモデラーがいずれにせよ従うワークフローを**固定化する**ことにあります。**まず導出し、次に翻訳し、段階的に検証する**——こうしてタイミングの誤り、命名の衝突、定常状態の代数ミスを、誤った結果になる前に食い止めます。具体的には、次のものを提供します。
>
> - **白紙ではなく、検証済みの出発点から** ——内蔵の二層リファレンスライブラリ（経済構造を担う 149 本の MMB 再現モデル + Dynare の書き方を担う 89 本の Pfeifer 例）により、すべてのモデルが動作するコードから出発します。
> - **使うほど鋭くなるツール** ——完成したモデルはすべてアーカイブされ、次回以降のタスクで参照されます。遭遇したバグはすべて修正法とともに記録され、同じ罠を二度デバッグすることはありません。
> - **論文にそのまま使える出力** ——すべての IRF は出版品質のベクター図として納品され、原稿にそのまま貼り込めます。
>
> タイミングの落とし穴なし。隠れたエラーなし。白紙なし。
>
> *マクロ経済の世界をお楽しみください。*

## 動作要件

| やりたいこと                                       | 必要なもの                                                              |
| -------------------------------------------------- | ----------------------------------------------------------------------- |
| `.mod` ファイルを**書く / 編集する / 検査する**    | [Claude Code](https://docs.claude.com/en/docs/claude-code/overview) のみ |
| 生成された `.mod` を自分で**実行する**             | 加えて [Dynare 7.1](https://www.dynare.org/) + MATLAB（または Octave）   |
| **自動実行とエラーの反復修正**をさせる            | さらに MATLAB MCP サーバーを VSCode に追加                              |

> MATLAB なしでも使えます。モデルの作成やエラーの検査はできますが、Dynare の実行は代行しません。
>
> 具体的な論文を提示すると、応答時間とトークン消費を大幅に削減できます。
>
> 🔌 **MCP は初めてですか？** MATLAB（や Stata）を Claude Code に接続したことがない場合は、ゼロから始める初心者向けガイドに従ってください: **[MATLAB / Stata を Claude Code に接続する（MCP セットアップガイド）](./docs/mcp-setup-guide_ja.md)**。事前設定なしを前提に、Claude Code のインストール、上記の自動実行・デバッグループを支える MATLAB MCP の接続、そして Stata MCP までをカバーします。

## インストール（約 1 分）

まず Claude Code をインストールします。[公式インストールガイド](https://docs.claude.com/en/docs/claude-code/overview)を参照してください。一般的な方法は `npm install -g @anthropic-ai/claude-code` です。その後:

1. ターミナルで任意のプロジェクトディレクトリに入り、`claude` を実行して Claude Code を起動します。
2. Claude Code の会話内で、次の 2 つのコマンドを**1 行ずつ**貼り付けます。

   ```text
   /plugin marketplace add EconSolider/dynare-copilot
   /plugin install dynare-copilot@econsolider-skills
   ```

   1 行目はこのリポジトリをプラグインマーケットプレイスとして登録し、2 行目が `dynare-copilot` プラグインをインストールします。
3. 完了です。`/` を入力してメニューに dynare-copilot skill が表示されれば、インストール成功です。

## アップデート

インストール後に最新版を取得するには、Claude Code で次の 2 つのコマンドを**1 行ずつ**実行します。

```text
/plugin marketplace update econsolider-skills
/plugin update dynare-copilot@econsolider-skills
```

1 行目はマーケットプレイスのメタデータをリポジトリから更新し、2 行目はインストール済みプラグインを最新リリースの `version` にアップグレードします。すぐに反映されない場合は Claude Code を再起動してください。

### 自動アップデートを有効にする（推奨）

サードパーティ製マーケットプレイスは自動アップデートが**既定でオフ**です。そのため、有効化しない限り毎回上記の 2 つのコマンドが必要になります。起動時に自動でこのマーケットプレイスを更新しプラグインをアップグレードさせるには:

1. `/plugin` を実行してプラグインマネージャーを開きます。
2. **Marketplaces** タブに移動し、`econsolider-skills` を選択します。
3. **Enable auto-update** を選びます。

設定後は、Claude Code を起動するたびに自動で最新版を取得します。更新が適用されると `/reload-plugins` の実行を促されます。この設定は一度だけで済みます。

## アンインストール

プラグインを削除するには、Claude Code で次を実行します。

```text
/plugin uninstall dynare-copilot@econsolider-skills
```

これはマーケットプレイスの登録を残したままプラグインのみを削除するので、再追加なしで再インストールやアップデートができます。

> 手動インストール（下記「上級者向け」の方法）の場合、アンインストールするプラグインはありません。コピーしたディレクトリを削除するだけです: `rm -rf ~/.claude/skills/dynare-copilot`（またはプロジェクト内の `.claude/skills/dynare-copilot/`）。

## クイックスタート

インストール後は、Claude Code の中で**中国語・英語・日本語でそのままタスクを記述**してください。プラグインは自動で有効になります。

- 「この FOC のセットを Dynare コードに変換して: `c^-sigma = beta*E[c(+1)^-sigma*(r(+1)+1-delta)]` ……」
- 「Smets-Wouters 2007 を四半期キャリブレーションで再現し、IRF を実行して。」
- 「私のニューケインジアンモデルに名目金利のゼロ下限（ZLB）を OccBin で追加して。」
- 「私の mod が `Blanchard-Kahn conditions are not satisfied` を出す。タイミングを確認して。」
- 「これらの変数の IRF をジャーナル品質の図にして、ベースライン vs 高粘着性の 2 シナリオを比較し、PDF で出力して。」

`/dynare-copilot:dynare-copilot` で手動呼び出しもできます。

この skill は `references/` 配下に 2 つのリファレンスライブラリ——経済構造のための 149 本の MMB 再現モデルと、Dynare 構文のための 89 本の Pfeifer 例——に加え、増え続けるあなた自身のモデルアーカイブをバンドルしています。モデル作成を依頼されると、まずこれらのローカルライブラリを検索し、どちらもカバーしない論文固有の詳細についてのみウェブ検索にフォールバックします。検索ロジックは[仕組み](#仕組み)を、各ライブラリの内容は[リポジトリ構成](#リポジトリ構成)を参照してください。

## 仕組み

記憶だけで書くことはありません。固定したワークフローに従います。

1. **まず確認、それから記述**: コーディング前に、**方程式の構造を変える可能性のあるモデリングの選択**（どのエージェントを含むか、労働供給は同質か異質か、資本を含むか、市場構造など）の承認を求めます。次に、最適化問題、FOC、定常状態の解、タイミングを網羅した構造化された導出ファイルを作成します。あなたの確認後にのみコードを書きます。この 2 つの一時停止は、後で大きなコードブロックを作り直すのではなく、誤りが起きる前に捉えるためのものです。
2. **段階的構築**: 変数宣言、方程式、定常状態、ショック、実験を段階ごとに記述します。各段階が動作してから次へ進みます。
3. **非線形優先**: 既定では、線形化システムを手で導出する代わりに、元の非線形方程式系を書いて Dynare に展開を任せます。手動の線形化は隠れた誤りのよくある原因です。
4. **実行-デバッグループ**: MATLAB MCP に接続されると、Dynare を自動実行し、エラーを読み、最小限の修正を加えて再実行します。既知の罠をまとめたバンドル済みの**エラーログ**（`known-issues.md`）をまず参照し、新たに解決したエラーは書き戻します——同じ罠を二度ゼロからデバッグすることはありません。
5. **ローカル二重ライブラリ検索**: モデルを書く前に、2 つの独立したローカルライブラリを検索します。

   - **モデルリファレンスライブラリ**（`catalog.csv`、149 本の MMB/rep-mmb モデル）: 「この経済メカニズムはどう構成すべきか」——FOC パターン、タイミング規約、キャリブレーション——に答えます。
   - **プログラミングロジックライブラリ**（`catalog-code.csv`、89 本の Johannes Pfeifer 例: 41 本の DSGE_mod + 48 本の *Advanced Dynare* コース）: 「この Dynare ブロックはどう書くか」——コマンド構文、`steadystate.m` インターフェース、`discretionary_policy`、`lmmcp`、厚生ブロック、ニュースショック、高次手法——に答えます。

   両方のローカルライブラリの後に、過去のセッションで構築した**個人のモデルアーカイブ**（`model-archive-catalog.csv`）を確認し、ライブラリがカバーしない論文固有の詳細（正確なキャリブレーション目標、導出手順）についてのみウェブ検索にフォールバックします。DSGE_mod は完全にローカルにバンドルされており、ウェブ取得は不要です。モデルアーカイブはモデリングタスクの終わりごとに自動的に増えます——最終的な `.mod` と導出ファイルが `references/model-archive/` にアーカイブされます。線形化済みのリファレンスモデルは方程式の内容、タイミング、キャリブレーションのためにのみ使われ、そのままコピーされることはありません。
6. **遅いモデルでの効率的な反復**: 解くのに高コストなモデル（異質性 / HANK、推定、高次）では、一度きりの求解と安価な図の微調整を分離します——結果をキャッシュすることで、再作図、再正規化、多モデル比較（例: HANK vs RANK）が決してモデルを解き直しません。既知の解析的ベンチマークが、図を信頼する前に出力され確認され、静かな正規化 / スケーリングの誤りを捉えます。

## 出力構成

モデリングタスクが完了すると、作業ディレクトリにはワンクリックで再実行可能な、自己完結した成果物一式が残ります。

```text
<model>_derivation.md   # 導出: 最適化問題 → FOC → 定常状態の解 → タイミング（コードの前にあなたの承認用に納品）
<model>.mod             # モデル本体: 変数 / パラメータ宣言、方程式、定常状態、ショック、実験コマンド
<model>_steadystate.m   # 外部定常状態ファイル（定常状態が数値求解を要する場合のみ。解析的定常状態は .mod の steady_state_model ブロックに入るため、このファイルは存在しない）
run_<model>.m           # 実行スクリプト: 自己完結のワンクリック再実行（addpath + cd + dynare + 整合性チェック + 作図スクリプト呼び出し + oo_ キャッシュ）。Dynare パスの 1 行を自分のマシン用に変えて実行
plot_irfs_pub.m         # 作図スクリプト: 出版品質の IRF（インパルス応答）ベクター図
plot_series_pub.m       # 作図スクリプト: 時系列 / シミュレーション経路 / 完全予見の遷移経路
fig_*.pdf               # 出版品質のベクター図（論文にそのまま貼り込める）
<model>_oo.mat          # 求解結果のキャッシュ（実行スクリプトが生成する、再生成可能な中間物）
```

毎回すべてのファイルが現れるわけではありません——出力はタスクに依存します。

- **導出ファイル**は、非標準メカニズム（TANK / HANK、金融摩擦、開放経済、最適政策、OccBin）を含む場合や、方程式の整合性チェックが必要な場合にのみ作成されます。教科書的な標準モデル（素の RBC、3 方程式 NK）ではスキップされます。
- **外部定常状態ファイル**は、定常状態が閉形式を持たず数値的に解く必要がある場合にのみ現れます。解析的な定常状態は直接 `.mod` に入ります。
- **作図スクリプトと図**は出力タイプで選ばれます。IRF は `plot_irfs_pub` を、シミュレーション系列 / 遷移経路は `plot_series_pub` を呼びます。図のないタスク（純粋な定常状態チェック、識別）ではスキップされます。
- **実行スクリプト**は既定で作成され、各部品をワンクリック再実行に配線し、**作図スクリプトを自動的に呼び出します**（もう手動で作図を実行する必要はありません）。プロジェクトに既存のトップレベル `main` がある場合は、新規ファイルではなくその `main` に求解と作図が配線されます。

クリーンアップ時には Dynare が自動生成した成果物（`+<model>/`、`Output/`、`*_results.mat`、`.log` など）のみを削除します。上記の成果物とアップロードしたデータは常に保持されます。

## 上級者向け

<details>
<summary>マーケットプレイスを使わない手動インストール</summary>

skill ディレクトリを個人の skills ディレクトリにコピーするだけです。プラグインマーケットプレイスは不要です。

```bash
cp -r plugins/dynare-copilot/skills/dynare-copilot ~/.claude/skills/
```

`~/.claude/skills/` を初めて作成した後は Claude Code を再起動してください。単一プロジェクトのみで使うには、そのプロジェクトの `.claude/skills/dynare-copilot/` に置きます。

</details>

<details>
<summary>ローカル検証とアップデート</summary>

```bash
claude plugin validate .                      # marketplace.json を検証
claude plugin validate ./plugins/dynare-copilot   # plugin.json と skill メタデータを検証
```

アップデートを公開するにはコミットをプッシュします。ユーザーは `/plugin marketplace update` で最新版を取得できます。リリースごとに `plugin.json` の `version` をインクリメントするのを忘れないでください。

</details>

<details>
<summary>実行時パス設定</summary>

自動実行に使う Dynare / MATLAB のパスは `references/workflow-detail.md` で設定します。既定の例は `C:\dynare\7.1` です。実際のインストールに合わせて変更してください。

</details>
